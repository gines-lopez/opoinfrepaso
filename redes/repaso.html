    Apuntes de Redes para Oposición¶

    1. VLSM y Routing¶

    VLSM (Variable Length Subnet Masking)¶

    ¿Qué es?¶

    Permite usar máscaras de subred de distinta longitud en una misma red, optimizando el uso de direcciones IP.

    Pasos para resolver un problema de VLSM:¶

    1. Listar subredes necesarias con el número de hosts por subred.
    2. Ordenar de mayor a menor según el número de hosts.
    3. Calcular la máscara adecuada para cada subred: - Fórmula: ( 2^n - 2 \geq \text{nº de hosts} )
    4. Asignar rangos de IPs, empezando por la red base.
    5. Asegurar que no haya solapamiento entre subredes.

    Routing¶

    Pasos para resolver ejercicios de Routing:¶

    1. Identificar redes conectadas directamente a cada router.
    2. Determinar qué redes no conoce el router y qué routers sí las conocen.
    3. Decidir entre rutas estáticas o dinámicas.
    4. Configurar las rutas estáticas con el comando adecuado (en Cisco, por ejemplo): 
      ip route [destino] [máscara] [siguiente salto]
    5. En caso de routing dinámico, configurar protocolo (RIP, OSPF, etc.).

    2. ARP (Address Resolution Protocol)¶

    ¿Qué hace?¶

    Convierte direcciones IP en direcciones MAC.

    Funcionamiento básico:¶

    1. Host A quiere comunicarse con IP B.
    2. Envía una solicitud ARP (broadcast) preguntando: ¿Quién tiene la IP B?
    3. El host B responde con su dirección MAC (unicast).
    4. Host A guarda esta información en su tabla ARP.

    Tabla ARP:¶

    IP MAC Address Estado
    192.168.1.10 00:1A:2B:3C:4D:5E Dinámico

    3. VLANs (Virtual LANs)¶

    ¿Qué es?¶

    Separan dominios de broadcast lógicamente, aunque estén en la misma red física.

    Beneficios:¶

    • Aislamiento de tráfico
    • Mejora de seguridad y rendimiento
    • Gestión centralizada

    Tipos:¶

    • Estáticas (puerto asignado a VLAN)
    • Dinámicas (basadas en MAC, protocolo, etc.)

    Trunk vs Access:¶

    • Access port: conecta host final, pertenece a una sola VLAN.
    • Trunk port: transporta varias VLANs (con etiquetas 802.1Q).

    Comandos Cisco:¶

    Switch(config)# interface fa0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

    4. IPv6 (con UEFI)¶

    IPv6 Básico:¶

    • 128 bits
    • Ejemplo: 2001:db8:85a3::8a2e:370:7334

    Tipos:¶

    • Unicast
    • Multicast
    • Anycast

    Estructura:¶

    • Prefijo de red /64
    • Interface ID /64

    Link-Local:¶

    • Rango: FE80::/10
    • Usada para funciones internas como NDP

    SLAAC:¶

    • Configuración automática de IP basada en prefijo del router

    UEFI y IPv6:¶

    • UEFI reemplaza BIOS
    • Permite arranque por red con IPv6 (PXE IPv6)

    5. Otros ejercicios frecuentes¶

    Ejercicio de Subredes:¶

    Dada la red 192.168.10.0/24, divide en 4 subredes: - Cada subred: /26 - Rango 1: 192.168.10.0 - 63
    - Rango 2: 192.168.10.64 - 127
    - Rango 3: 192.168.10.128 - 191
    - Rango 4: 192.168.10.192 - 255

    Ejercicio de NAT:¶

    Traducción de IPs internas a una IP pública mediante PAT: 

    Internas: 192.168.1.10, .11, .12
IP Pública: 80.80.80.1
Puerto: PAT permite multiplexación

    Ejercicio de VLAN:¶

    Crear VLANs 10 y 20. Asignar puerto fa0/1 a VLAN 10: 

    Switch(config)# interface fa0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
